# LangChain 核心介绍
LangChain 是一款开源的 Python/JavaScript 框架，核心定位是**降低大语言模型（LLM）应用开发门槛**，通过模块化设计将 LLM、外部工具、数据来源、上下文记忆等能力串联，让开发者无需从零搭建复杂逻辑，就能快速构建智能体（Agent）、检索增强生成（RAG）、多轮对话机器人等进阶 LLM 应用。截至 2026 年，LangChain 已成为 LLM 应用开发的主流工具，适配 OpenAI、Anthropic、国产大模型等几乎所有主流 LLM 生态。

## 核心设计理念
LangChain 的核心思路是“模块化+可组合”：将 LLM 应用的核心能力拆解为独立模块，开发者可按需组合模块，像搭积木一样构建应用。比如，给基础 LLM 搭配“搜索工具”和“思考逻辑”，就能实现自主查询最新信息的智能体；搭配“文档加载器”和“向量数据库”，就能实现基于私有文档的精准问答。这种设计既降低了新手的学习成本，也让资深开发者能灵活定制复杂流程。

## 核心功能模块
LangChain 的核心能力集中在 7 个核心模块，覆盖 LLM 应用开发的全流程：
1. **Models**：统一的模型调用层，支持各类 LLM（如 GPT-4o、Claude 3）和聊天模型（ChatOpenAI），封装不同模型的 API 差异，开发者用统一接口调用不同模型；
2. **Prompts**：提示词模板管理模块，支持自定义模板、从 LangChain Hub 拉取官方预定义模板（如 ReAct 智能体模板），解决提示词格式不统一、复用难的问题；
3. **Tools**：外部工具集成层，内置 Tavily 搜索、PythonREPL 代码执行、数据库查询、API 调用等数十种工具，让 LLM 突破“仅靠训练数据回答”的限制；
4. **Agents**：智能体核心模块，实现“思考→选工具→执行→总结”的自主决策逻辑（经典 ReAct 模式），是 LangChain 最具特色的能力；
5. **Memory**：上下文记忆模块，支持短期（会话级）和长期（持久化）记忆，让 LLM 能记住多轮对话内容，解决“健忘”问题；
6. **Chains**：链式调用模块，串联多个操作步骤（如“加载文档→拆分文本→检索相关内容→生成回答”），实现复杂流程自动化；
7. **Document Loaders**：文档加载模块，支持 PDF、Excel、网页、Markdown 等几乎所有格式的文档解析，为 RAG 场景提供数据输入。

## 核心应用场景
LangChain 最核心的两大落地场景是：
1. **智能体（Agent）**：让 LLM 从“被动回答”升级为“主动解决问题”。比如，给 Agent 配置搜索工具后，它能自主查询“2026 年上海最新房价”这类时效性问题，而非依赖过时的训练数据；配置 PythonREPL 工具后，能自主执行代码完成数学计算、数据处理。
2. **检索增强生成（RAG）**：解决 LLM“知识过时”“胡说八道”的痛点。通过加载本地文档（如企业手册、PDF 报告），将文档拆分后存入向量数据库，用户提问时先检索相关文档片段，再传给 LLM 生成基于文档的精准回答，广泛应用于企业知识库问答、文档解析场景。

## 版本与生态
LangChain 分为旧版（<0.1.0）和新版（≥0.1.0）：旧版功能集中在 `langchain` 主包，新版做了模块化拆分（`langchain-core` 核心抽象、`langchain-community` 第三方集成、`langchain-experimental` 实验性功能），更符合工程化开发规范。配套生态包括 LangChain Hub（提示词模板库）、LangSmith（调试监控平台）、LangServe（应用部署工具），形成“开发-调试-部署”的完整闭环。

对新手而言，LangChain 降低了 LLM 应用的开发门槛，无需深入理解模型底层原理，就能聚焦业务逻辑；对企业而言，其模块化设计和丰富的集成能力，能快速适配不同场景的定制化需求，是当前 LLM 应用落地的核心工具之一。
